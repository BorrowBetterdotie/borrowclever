# /data

Source-of-truth JSON for all rate data. Edit here, run `node build.js`, then diff and commit.

## Files

| File | Purpose |
|------|---------|
| `loans.json` | All personal loan products, sorted by APR ascending |
| `cards.json` | Credit card products (placeholder — not yet wired into build) |

## Loan record fields

| Field | Type | Notes |
|-------|------|-------|
| `id` | string | Stable slug, e.g. `ptsb-personal` |
| `lender` | string | Display name |
| `product` | string | Product name |
| `icon` | string | Abbreviation shown in icon circles |
| `iconCls` | string | CSS class for icon colour |
| `apr` | number | Effective annual rate (%) — this is the key comparison figure |
| `rateType` | string | `"Fixed"` or `"Variable"` |
| `typeNote` | string | Short feature note shown on homepage card, e.g. `"No conditions"` |
| `green` | boolean | Marks green/energy loan rows |
| `seai` | boolean | Requires SEAI home energy grant |
| `creditUnion` | boolean | Credit union product |
| `homepage` | boolean | **Featured on the homepage mini-table** — see below |
| `notes` | string | Footnote text in the full loans table |
| `badgeCheapestOverall` | boolean | Shows "Cheapest overall" badge |
| `badgeBestStandard` | boolean | Shows "Best standard rate" badge |

## Homepage hero generation

`index.html`'s hero panel and "Switch and save" callout are generated by `build.js` — **do not edit those sections by hand**.

- The featured set is the four loans with `"homepage": true` in `loans.json`: currently `ptsb-personal`, `boi-personal`, `anpost-personal`, `aib-personal`.
- The saving figure derives automatically: `totalInterest(aib-personal) − totalInterest(bestFeatured)`, using the effective monthly rate formula `i = (1 + APR/100)^(1/12) − 1` over 60 months on €10,000.
- Interest figures are not stored — they are recomputed on every build.

## Fortnightly update workflow

1. Edit the relevant `apr` (and any other fields) in `loans.json`.
2. Run `node build.js` from the repo root.
3. Review the diff — `index.html` hero numbers and the loans table will both update.
4. Commit and open a PR.
